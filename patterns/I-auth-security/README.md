# 🅸 認證 / 安全

## 共通模式
跨系統的「**你是誰、你能做什麼**」：身分驗證（authentication）、授權（authorization）、單一登入（SSO）與權杖（token）的簽發、驗證、撤銷。重心在**信任邊界**、**最小權限**與**權杖生命週期**的正確設計。

## 可複用心法
- **authn vs authz**：認證確認身分（你是誰），授權決定權限（你能做什麼）；OAuth 管授權、OIDC 在其上做認證。
- **權杖類型**：自包含 **JWT**（無狀態、可水平擴展驗證、但難即時撤銷）vs 不透明 token + **introspection**（可即時撤銷、但每次要查中心）。
- **授權碼流程**：授權碼**單次使用**、綁定 client/redirect/scope；**PKCE** 防授權碼攔截。
- **撤銷難題**：無狀態 JWT 無法直接作廢 → 用「**短期 access token + 撤銷名單 / refresh 輪替**」折衷。
- **金鑰管理**：簽章金鑰輪替（JWKS）、對稱 HS256 vs 非對稱 RS256（公鑰可公開驗章）。
- **縱深防禦**：HTTPS、CSRF/重放防護、常數時間比對、限流與審計。
