# 🅰️ 通知 / Fan-out 扇出

## 共通模式
事件發生（地震偵測 / 價格變動 / 系統事件）→ 需要**可靠地、大規模地**把通知投遞給大量訂閱者。

## 可複用心法
- **解耦**：產生事件的服務 ≠ 投遞通知的服務，中間用 **訊息佇列**（Kafka / SQS）。
- **Fan-out**：一個事件 → N 個接收者。讀放大時用「fan-out on write」或「fan-out on read」取捨。
- **投遞保證**：at-least-once + 冪等（idempotency key）；retry 用**指數退避 + 抖動**；多次失敗進 **DLQ（死信佇列）**。
- **分區擴展**：依 user_id / region 分區並行投遞。
- **去重 / 限流**：避免重複通知轟炸；per-user rate limit。
