# 🅲 一致性 / 交易

## 共通模式
涉及金錢 / 庫存 / 名額，**不能超賣、不能雙花**，需要強一致與正確的並發控制。

## 可複用心法
- **ACID 交易**：關鍵路徑用關聯式資料庫交易；理解隔離級別（避免幻讀/寫偏斜）。
- **並發控制**：
  - **悲觀鎖** `SELECT ... FOR UPDATE`（高衝突）
  - **樂觀鎖** version / CAS（低衝突，高吞吐）
- **防超賣**：庫存遞減用條件更新 `UPDATE ... WHERE stock > 0`。
- **跨服務一致性**：**Saga**（補償交易）或 **2PC**；冪等與對帳。
- **撮合 / 結算**：order book 單執行緒撮合 + 事件溯源（event sourcing）保證可重放。
