# 🅻 圖 / 大規模演算法

## 共通模式
這組是「**演算法密集 + 大規模資料**」的題型：把問題建模成圖或索引，用對的演算法在海量資料上算出結果（最短路徑、相關性排序、權威分數、最小轉帳），再靠分片與快取擴展。重心在**資料結構選擇**與**離線預處理 vs 線上查詢**的取捨。

## 可複用心法
- **建圖建索引**：把領域問題轉成圖（道路網、連結圖、欠款圖）或倒排索引，演算法就現成可用。
- **預處理換查詢速度**：離線重活（PageRank 冪次迭代、CH 捷徑、相似度矩陣）先算好，線上只做輕量查詢；空間/新鮮度換延遲。
- **分片 + scatter-gather**：索引依雜湊分片，查詢平行打所有 shard 取局部結果再合併排序；用全域統計（df/avgdl）確保跨片可比。
- **啟發式剪枝**：A* 用可採納的啟發式（直線距離）朝目標收斂，比 Dijkstra 探索更少節點。
- **機率型結構省記憶體**：bloom filter 做 URL 去重，犧牲少量偽陽性換大量空間。
- **禮貌與去重**：爬蟲依 domain 分片集中 politeness，BFS frontier + 去重避免重複與陷阱。
- **守恆與化簡**：帳務淨額恆為 0；債務圖用貪婪（最大債主↔最大欠款）化簡成最少轉帳。
