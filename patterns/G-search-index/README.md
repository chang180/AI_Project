# 🅶 搜尋 / 索引 / 地理空間

## 共通模式
給定大量資料，要能依**前綴、關鍵字或位置**快速查到 Top-K 結果。重心在「用對的資料結構建索引」與「讀多寫少的索引更新策略」。

## 可複用心法
- **Trie（字典樹）**：前綴查詢、自動完成；在節點預存 Top-K，查詢即 O(前綴長度)。
- **倒排索引**：詞 → 文件清單；全文搜尋、相關性排序（TF-IDF / BM25）的基礎。
- **空間索引**：Geohash（前綴分桶，查中心格 + 8 鄰格解邊界）/ QuadTree / S2；半徑查詢「粗篩 + haversine 精算」兩階段。
- **索引更新**：查詢日誌 → 聚合計頻 → 近即時或離線批次重建索引；讀路徑靠快取與讀副本擴展。
- **Top-K 排序**：頻率 / 距離 / 評分；分頁與屬性過濾結合。
