# 🅵 社交 / Feed 分發

## 共通模式
使用者產生內容（貼文）→ 需要把內容**分發到大量追蹤者的時間線**，並在讀取時快速組裝、排序。本質是 fan-out 的進階版：不只是「投遞通知」，而是要維護每個人持續更新、可分頁的動態流。

## 可複用心法
- **推模型（fan-out on write）**：發文時就寫入每個追蹤者的 feed 快取。讀快、寫放大；適合粉絲數少的一般使用者。
- **拉模型（fan-out on read）**：讀 feed 時即時聚合所有追蹤對象的最新貼文。寫便宜、讀貴；適合追蹤對象變動大的情境。
- **混合（hybrid）解名人問題**：粉絲數爆量的名人不推（避免一次寫千萬份），改在讀時拉；一般人照推 → 讀時合併兩者。
- **Feed 快取只存 id**：feed 存貼文 id 清單，讀時再 hydrate 內容，省空間、好失效。
- **排序與分頁**：時間序 / 排序分數；用游標分頁（cursor）避免 offset 漂移。
